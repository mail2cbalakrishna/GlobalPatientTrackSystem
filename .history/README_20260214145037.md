### Microservices Architecture
- **API Gateway** (Port: 8080) - Central entry point with routing and security
- **Auth Service** (Port: 8081) - OAuth2.0 authentication and JWT token management
- **User Data Service** (Port: 8082) - User management and role-based access control
- **Doctor Service** (Port: 8083) - Doctor portal and patient management
- **Patient Service** (Port: 8084) - Patient portal and health data management
- **Admin Service** (Port: 8085) - Administrative portal with full system access
- **Lab Service** (Port: 8086) - Lab test management and reports
- **Service Discovery** (Port: 8761) - Eureka Server for service registration
- **Config Server** (Port: 8888) - Centralized configuration management

### Frontend UI Architecture

The frontend is built with **React** and **Tailwind CSS**, providing three specialized portals:

#### 1. **Admin Portal**
- **URL**: `http://localhost:8080/admin/`
- **Features**:
  - View all patients in the system
  - View all registered doctors
  - Manage user accounts and permissions
  - System analytics and reports
  - User role management (Assign ADMIN, DOCTOR, PATIENT roles)
  - Audit logs and system monitoring

#### 2. **Doctor Portal**
- **URL**: `http://localhost:8080/doctor/`
- **Features**:
  - View assigned patients
  - Update patient health records
  - Search and consult with other doctors
  - Manage lab test orders
  - View patient history and medical records
  - Generate medical reports

#### 3. **Patient Portal**
- **URL**: `http://localhost:8080/patient/`
- **Features**:
  - View personal health records
  - Access medical history
  - View lab test results
  - Schedule appointments
  - Communicate with assigned doctors
  - Download health reports

### Key Features
- **Three distinct portals**: Admin, Doctor, and Patient with role-based access
- **Responsive UI**: Tailwind CSS for modern, responsive design
- **OAuth2.0 Security**: JWT token-based authentication and authorization
- **Microservices Architecture**: Scalable and maintainable service design
- **Service Discovery**: Automatic service registration and discovery
- **Containerized Deployment**: Docker and Docker Compose support
- **Real-time Data Sync**: Context API for state management

### User Roles & Permissions
- **ADMIN**: Full system access - view all patients and doctors, manage users, system monitoring
- **DOCTOR**: Access to assigned patients, search other doctors by name/ID, manage medical records
- **PATIENT**: Access only to personal health data and history, view reports

## Getting Started

### Prerequisites
- Java 17
- Maven 3.6+
- Docker & Docker Compose
- MySQL 8.0+

### Running the Application
1. Clone the repository
2. Start the infrastructure services:
   ```bash
   docker-compose up -d mysql
   ```
3. Start the services in order:
   ```bash
   # Service Discovery
   cd service-discovery && mvn spring-boot:run
   
   # Config Server
   cd config-server && mvn spring-boot:run
   
   # Auth Service
   cd auth-service && mvn spring-boot:run
   
   # User Data Service
   cd user-data-service && mvn spring-boot:run
   
   # Business Services
   cd doctor-service && mvn spring-boot:run
   cd patient-service && mvn spring-boot:run
   cd admin-service && mvn spring-boot:run
   
   # API Gateway
   cd api-gateway && mvn spring-boot:run
   ```

### API Endpoints
- **API Gateway**: http://localhost:8080
- **Admin Portal**: http://localhost:8080/admin/**
- **Doctor Portal**: http://localhost:8080/doctor/**
- **Patient Portal**: http://localhost:8080/patient/**

## Database Schema
- **USERSDATA**: User credentials and role management
- **PATIENTS**: Patient demographic and health information
- **DOCTORS**: Doctor profiles and specializations
- **APPOINTMENTS**: Appointment scheduling and management
- **LAB_TESTS**: Lab test orders and results
- **MEDICAL_RECORDS**: Patient medical history and records
- **AUDIT_LOGS**: System activity and access logs

## Deployment & Infrastructure

### Docker Deployment
All services are containerized and can be deployed using Docker Compose:
```bash
docker-compose up -d
```

### Service Communication
- Services communicate via REST APIs
- Eureka Service Discovery enables dynamic service lookup
- Config Server manages centralized configuration
- API Gateway provides single entry point
