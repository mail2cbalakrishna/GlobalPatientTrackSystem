### Microservices Architecture
- **API Gateway** (Port: 8080) - Central entry point with routing and security
- **Auth Service** (Port: 8081) - OAuth2.0 authentication and JWT token management
- **User Data Service** (Port: 8082) - User management and role-based access control
- **Doctor Service** (Port: 8083) - Doctor portal and patient management
- **Patient Service** (Port: 8084) - Patient portal and health data management
- **Admin Service** (Port: 8085) - Administrative portal with full system access
- **Lab Service** (Port: 8086) - Lab test management and reports
- **Service Discovery** (Port: 8761) - Eureka Server for service registration
- **Config Server** (Port: 8888) - Centralized configuration management

### Key Features
- **Three distinct portals**: Admin, Doctor, and Patient with role-based access
- **OAuth2.0 Security**: JWT token-based authentication and authorization
- **Microservices Architecture**: Scalable and maintainable service design
- **Service Discovery**: Automatic service registration and discovery
- **Containerized Deployment**: Docker and Docker Compose support

### User Roles & Permissions
- **ADMIN**: Full system access - view all patients and doctors
- **DOCTOR**: Access to assigned patients, search other doctors by name/ID
- **PATIENT**: Access only to personal health data and history

## Getting Started

### Prerequisites
- Java 17
- Maven 3.6+
- Docker & Docker Compose
- MySQL 8.0+

### Running the Application
1. Clone the repository
2. Start the infrastructure services:
   ```bash
   docker-compose up -d mysql
   ```
3. Start the services in order:
   ```bash
   # Service Discovery
   cd service-discovery && mvn spring-boot:run
   
   # Config Server
   cd config-server && mvn spring-boot:run
   
   # Auth Service
   cd auth-service && mvn spring-boot:run
   
   # User Data Service
   cd user-data-service && mvn spring-boot:run
   
   # Business Services
   cd doctor-service && mvn spring-boot:run
   cd patient-service && mvn spring-boot:run
   cd admin-service && mvn spring-boot:run
   
   # API Gateway
   cd api-gateway && mvn spring-boot:run
   ```

### API Endpoints
- **API Gateway**: http://localhost:8080
- **Admin Portal**: http://localhost:8080/admin/**
- **Doctor Portal**: http://localhost:8080/doctor/**
- **Patient Portal**: http://localhost:8080/patient/**

## Database Schema

- JWT token validation across services
- Role-based method security
- Secure inter-service communication

## Development Guidelines
- Follow microservices best practices
- Implement proper error handling and logging
- Use DTOs for data transfer between services
- Maintain service independence and loose coupling
